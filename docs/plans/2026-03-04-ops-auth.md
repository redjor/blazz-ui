# Ops Auth — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ajouter l'authentification Google OAuth à `apps/ops` via `@convex-dev/auth`, avec middleware Next.js qui redirige les utilisateurs non connectés vers `/login`.

**Architecture:** `@convex-dev/auth` gère l'intégralité du flow OAuth côté Convex (tokens JWT, callbacks HTTP, session). Le middleware Next.js vérifie le cookie JWT et redirige si absent. `ConvexAuthProvider` remplace `ConvexProvider` côté client.

**Tech Stack:** `@convex-dev/auth`, `@auth/core/providers/google`, Next.js 16 App Router middleware, Convex HTTP router

---

## Prérequis manuels (à faire avant de coder)

### 1. Créer un projet Google Cloud OAuth

1. Aller sur [console.cloud.google.com](https://console.cloud.google.com)
2. APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID
3. Application type: **Web application**
4. Authorized redirect URIs — ajouter :
   - `http://localhost:3120/api/auth/callback/google` (dev local)
   - `{CONVEX_SITE_URL}/api/auth/callback/google` (le site URL Convex — visible dans le dashboard Convex sous Settings → URL)
5. Copier le **Client ID** et **Client Secret**

### 2. Configurer les variables d'environnement Convex

Ces variables vont dans **Convex** (pas `.env.local`) car elles sont utilisées par les fonctions Convex :

```bash
cd apps/ops

# Générer la clé JWT (Ed25519)
npx @convex-dev/auth

# Ou manuellement :
npx convex env set AUTH_GOOGLE_ID "ton-client-id.apps.googleusercontent.com"
npx convex env set AUTH_GOOGLE_SECRET "ton-client-secret"
npx convex env set SITE_URL "http://localhost:3120"
# JWT_PRIVATE_KEY est généré par `npx @convex-dev/auth` — sinon :
npx convex env set JWT_PRIVATE_KEY "$(openssl genpkey -algorithm ed25519 2>/dev/null | openssl pkcs8 -topk8 -nocrypt -outform pem 2>/dev/null)"
```

## Task 3 : Ajouter les routes HTTP OAuth dans `http.ts`

`@convex-dev/auth` expose des routes HTTP pour gérer les callbacks OAuth. Il faut les enregistrer dans le router HTTP Convex.

**Files:**
- Modify: `apps/ops/convex/http.ts`

**Step 1: Modifier `http.ts`**

```ts
// apps/ops/convex/http.ts
import { httpRouter } from "convex/server"
import { httpAction } from "./_generated/server"
import { api } from "./_generated/api"
import { auth } from "./auth"

const http = httpRouter()

// Auth routes (Google OAuth callbacks)
auth.addHttpRoutes(http)

// Telegram webhook (inchangé)
http.route({
	path: "/telegram-webhook",
	method: "POST",
	handler: httpAction(async (ctx, request) => {
		const url = new URL(request.url)
		const secret = url.searchParams.get("secret")
		const expectedSecret = process.env.TELEGRAM_WEBHOOK_SECRET

		if (!expectedSecret || secret !== expectedSecret) {
			return new Response("Unauthorized", { status: 401 })
		}

		let body: {
			message?: {
				text?: string
				chat?: { id: number }
			}
		}
		try {
			body = await request.json()
		} catch {
			return new Response("Bad Request", { status: 400 })
		}

		const text = body?.message?.text
		if (!text) {
			return new Response("OK", { status: 200 })
		}

		await ctx.runMutation(api.todos.create, {
			text,
			status: "triage",
			source: "telegram",
		})

		return new Response("OK", { status: 200 })
	}),
})

export default http
```

## Task 7 : Créer le middleware de protection des routes

**Files:**
- Create: `apps/ops/middleware.ts`

**Step 1: Créer le middleware**

```ts
// apps/ops/middleware.ts
import {
	convexAuthNextjsMiddleware,
	createRouteMatcher,
	nextjsMiddlewareRedirect,
} from "@convex-dev/auth/nextjs/server"

const isPublicPage = createRouteMatcher(["/login"])

export default convexAuthNextjsMiddleware(async (request, { convexAuth }) => {
	if (!isPublicPage(request) && !(await convexAuth.isAuthenticated())) {
		return nextjsMiddlewareRedirect(request, "/login")
	}
})

export const config = {
	matcher: ["/((?!.*\\..*|_next).*)", "/", "/(api|trpc)(.*)"],
}
```
